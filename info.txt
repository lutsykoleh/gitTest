git revert - ця команда викорстовується для скасування змін комміта, вона створює комміт який значення минулого комміта перезаписує, що дає змогу скасувати комміт.
git reset - ця команда скидує стан гілки до вказаного коміту, і при цьому може видаляти коміти з історії. Ця команда корисна коли потрібно скинути локальні зміни до певного стану.
git branch -d - ця команда використовується для видалення гілки, але з додатковою перевіркою безпеки. Вона дозволить видалення лише тоді, коли зміни в гілці, яку ви намагаєтеся видалити, були повністю злиті з гілкою, на яку ви зараз переключені
git branch -D - ця команда є примусовою версією попередньої та дозволяє видалити будь-яку гілку, незалежно від її статусу злиття. Використання -D позбавляє захисту від втрати незлитих змін.
git rebase - ця команда використовується для того щоб об'єднати гілки, він бере всі комміти з гілки і додає їх у кінець вказаної гілки
git cherry-pick - ця команда перекидає один або декілька комітів в гілку з іншої гілки, для використання команди потрібен комміт хеш
